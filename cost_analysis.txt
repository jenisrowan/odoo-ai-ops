================================================================================
                     MONTHLY COST ANALYSIS - AWS INFRASTRUCTURE
                     Odoo AI Ops (us-east-1 Region Pricing)
================================================================================

Assumptions:
  • All prices in USD/month unless stated otherwise.
  • EC2 pricing uses On-Demand rates (no Reserved Instances or Savings Plans).
  • Odoo uses c6g.xlarge (4 vCPU, 8 GiB RAM).
  • PgBouncer uses m6g.medium (1 vCPU, 4 GiB RAM).
  • FastAPI Agent uses m6g.large (2 vCPU, 8 GiB RAM).
  • ClickHouse uses r6g.xlarge (4 vCPU, 32 GiB RAM).
  • RDS PostgreSQL uses db.m6g.xlarge (4 vCPU, 16 GiB RAM) Multi-AZ.
  • HA model: 1 active instance per service. 2nd instance boots on demand
    via ECS auto-scaling (not always-on) for Odoo, PgBouncer, and FastAPI.
  • VPC Endpoints: Gateway for S3 (free), Interface Endpoint for SQS (Amazon PrivateLink) across 2 AZs.
  • NAT Gateway is used for external API egress. SQS polling routes via the VPC Endpoint.
  • "Idle" = infrastructure running with zero order volume.
  • "Low" = ~100 flagged orders/day (webhook-triggered AI tasks).
  • "Growth" = ~1,000 flagged orders/day.
  • Order Risk Profile (of flagged orders):
      - No/Very Low Risk orders, and cheap orders (< $10) with Medium/High Risk, are bypassed and rejected automatically (0 AI tokens used).
      - 70% of flagged orders are Medium Risk (Claude 4.5 Haiku).
      - 30% of flagged orders are High Risk (Claude 4.6 Sonnet).

---  SECTION 3: SCALING PROJECTIONS  ------------------------------------------

Assumptions per flagged order:
  • 1 SQS message received + 1 SQS delete = 2 SQS requests
  • 1 API Gateway invocation (webhook ingestion)
  • 1 Webhook Ingest Lambda invocation
  • 1 Shopify GraphQL call (~20 KB response) -> via NAT
  • 1 Claude invocation (~2K in / ~1K out, 70% Haiku 4.5 / 30% Sonnet 4.6) -> via NAT
  • 1 Slack Block Kit push + 1 Slack approval callback -> via NAT
  • SQS traffic (~5 KB/order) routes via SQS Interface Endpoint
  • ~20 ElastiCache Serverless commands (state machine reads/writes)

  Valkey to ElastiCache Serverless completely removes state management overhead
  and memory contention from your application nodes.
